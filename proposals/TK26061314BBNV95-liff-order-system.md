## 一、我對這個需求的理解

貴公司希望讓客戶**透過 LINE 完成下單、查詢訂單、取得發票**，不必另外安裝 App、不必註冊帳號密碼。

核心要做的，是一套「**內嵌在 LINE 裡的訂單系統**」：

- 客戶在 LINE 圖文選單點一下 → 直接開啟下單頁（系統自動辨識是哪位客戶，免登入）
- 下單後可隨時在 LINE 裡查自己的訂單與狀態
- 訂單成立、出貨、發票開立時，系統主動用 LINE 訊息通知客戶
- 發票依法透過電子發票加值中心開立並寄送

技術上採用 **LIFF（LINE Front-end Framework）**——一個跑在 LINE App 內的網頁，最大的優勢是**免帳密、自動辨識用戶身分**，最適合傳統產業、長輩或司機等不擅長記帳密碼的使用情境。

## 二、系統架構

```
  LINE App
    └─ 圖文選單
         └─ LIFF 內嵌網頁（下單 / 訂單查詢 / 發票）
                    │  自動取得 LINE 身分，免登入
                    ▼
              訂單後端 API
                    │
        ┌───────────┼───────────────┐
        ▼           ▼               ▼
     訂單資料庫   LINE 推播      電子發票加值中心
                訂單/發票通知    （合規開立 + 寄送 + 載具）
```

- **前端（LIFF 網頁）**：下單、訂單查詢、發票三大功能頁，自動辨識 LINE 身分。
- **訂單後端**：訂單資料、狀態流程、供貴公司內部人員管理訂單的後台。
- **LINE 推播**：訂單成立、出貨、發票開立等關鍵節點主動通知客戶。
- **電子發票**：串接合規加值中心（綠界 / ezPay 等），不自行開立。

> 本系統將建構於提案方既有的多租戶服務平台之上，LINE 串接、訊息推播、客服機制皆有成熟基礎，可縮短開發時程、降低後續維運成本。

## 四、在報價前，需與貴公司確認的事項

以下幾點會直接影響系統設計與工作量，懇請先行確認：

1. **電子發票帳號**
   依台灣法規，電子發票須透過財政部電子發票整合服務平台，或經由加值服務中心（綠界 / ezPay 等）開立，**無法由系統自行產製**。貴公司是否已有加值中心帳號？若無，需先行申辦（涉及月費或交易抽成，由貴公司與加值中心簽約，此部分非開發費用範疇）。

2. **訂單的「品項」長相**
   是固定商品（如電商選購）？還是「運輸服務」（起迄點、車種、時間、噸數）？兩者的下單表單與資料設計差異很大，請提供 1～2 個實際下單範例。

3. **是否需要管理後台**
   貴公司出貨 / 內勤人員如何看單、改狀態？是純靠 LINE，還是需要一個網頁後台？（後台是常被低估的工作量，建議納入。）

4. **訂單量級與付款方式**
   每日約幾筆訂單？付款採線上金流、貨到付款、或月結？**若需線上付款（信用卡 / LINE Pay）為額外獨立模組**，需另行評估。

5. **客戶資料來源**
   現有客戶名單／編號如何對應到 LINE 帳號？是否需要首次綁定流程？
