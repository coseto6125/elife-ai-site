2我們對需求的理解

我們理解這是一個 AI 聚合聊天平台，核心在於對使用者隱藏多模型的存在，讓所有互動都呈現為單一且一致的人格。使用者送出問題後，後端的智慧路由機制會分析內容、為候選模型計算評分、挑選最適合的一個來回答，最後再以統一人格改寫輸出，使體驗連貫自然。

  • 路由是整個產品的關鍵，需要依問題類型自動在不同模型池之間做選擇，並兼顧成本與品質。
  • 三個方案分層管理，免費版控制成本只開放基礎模型池與每日額度，Premium 與 Ultra 逐步開放更高級的模型池、更高額度、檔案與 PDF 分析，以及影片生成。
  • 人格改寫需要過濾各模型的自我宣告與固定開場語，確保輸出風格一致，這是維持品牌人格的重要環節。
  • 圖片生成要求在可控成本下接近主流體驗，並支援多種風格用途；影片生成則作為 Ultra 的限定功能，受次數與長度限制。
  • 管理後台需要完整掌握使用量、各項成本、路由決策紀錄與訂閱收入，方便營運判斷。
  • 第一版聚焦聊天、路由、訂閱、圖片生成與管理後台，影片生成留待後續加入，這個分段我們認為相當合理。

3我們能提供的能力

我們具備從前端介面、後端服務到部署交付的整體開發能力，能依您規劃的架構落實這套聚合平台，並以 MVP 優先的方式分段交付。

  • 前端以 Next.js 與 React 搭配 TypeScript 開發，提供統一人格的聊天介面、訂閱頁與管理後台。
  • 後端以 Python 與 FastAPI 建置路由服務，串接多個模型並實作評分與選擇邏輯，搭配人格改寫的後處理流程。
  • 資料層採 PostgreSQL 儲存核心資料、Redis 處理快取與額度控管、Qdrant 支援向量檢索需求。
  • 圖片與影片生成的串接、依方案的額度限制與優先級控管，我們都能依規格實作。
  • 管理後台呈現使用者數量、各模型用量、API 與圖片影片成本、路由決策紀錄與訂閱收入。
  • 交付方式涵蓋 Web 的 Docker 映像檔與 Nginx 部署版本，行動端可協助產出 iOS 的 IPA 與 Android 的 AAB，行動框架可依後續討論選定。
