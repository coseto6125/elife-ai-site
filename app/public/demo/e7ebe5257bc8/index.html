2我們對需求的理解

這是一個聚合多家模型的聊天平台。使用者面對的是單一統一人格，背後則由路由機制依問題內容自動挑選合適的模型回答，再經由統一人格改寫後輸出，使用者不會察覺底層模型的切換與差異。

  • 核心是路由邏輯與人格層的配合。路由負責在多個模型之間做選擇與評分，人格層負責把不同來源的回答收斂成一致的語氣與稱呼，並移除可辨識來源的開場語與身分宣告。
  • 三個訂閱層級（免費、Premium、Ultra）在可用模型池、每日圖片與影片額度、聊天額度與回應優先級上做出區隔，免費版以控制成本為前提，較高層級開放更大的模型池與更多功能。
  • 能力面涵蓋聊天、圖片生成、影片生成（限最高層級）、檔案與 PDF 處理，並需要一套管理後台來掌握使用量、各項成本與訂閱收入，以及路由的決策紀錄。
  • 交付目標包含 iOS、Android 與 Web 三端，最終需要可上架與可部署的產出。

3我們能提供的能力

我們可以從第一版可運作的範圍著手，先把核心流程與訂閱機制建立起來，再依實際使用情況擴充模型池與進階功能。

  • 路由與人格層：建立提問分析、模型評分與選擇、統一人格改寫的處理流程，並設定改寫規則以移除來源辨識資訊與制式開場語，使輸出維持一致風格。
  • 多模型串接：以可擴充的方式接入聊天模型，依層級劃分基礎與進階模型池，後續新增或替換模型不需更動整體架構。
  • 訂閱與額度：實作三個層級的權限劃分，包含每日聊天與圖片額度、影片產製的頻率與長度限制，以及回應優先級的差異。
  • 圖片與影片：串接圖片生成並支援多種風格輸出，影片生成依最高層級的頻率與長度限制提供文生與圖生影片。
  • 檔案處理：支援檔案上傳與 PDF 內容的解析與問答。
  • 管理後台：提供使用者數、各模型使用量、各項成本、路由決策紀錄與訂閱收入的檢視。
  • 多端交付：產出 iOS、Android 與 Web 三端的可上架與可部署版本。
關鍵相依：各模型與生成服務的接取需要有效的帳號與授權，使用所產生的呼叫費用由業主承擔。圖片與影片的實際品質與成本，受所選服務的能力與計價方式影響。
